# 질문 프롬프트: 스테이션 대시보드에서 DEV 버튼이 사라진 이유

아래 블록 전체를 다른 LLM 에이전트에게 전달해, **작업환경만 옮긴 뒤 스테이션에서 보이던 버튼이 새 환경에서 안 보이는 원인**을 정확히 찾고 해결 방법을 제시해 달라고 요청하세요.

---

## 복사용 프롬프트 (전체 복사 후 붙여넣기)

```
[상황]
- 프로젝트: Next.js + Tauri 프론트엔드, FastAPI 백엔드. 스테이션 대시보드 페이지가 있음.
- 이전 개발 환경에서는 스테이션 대시보드 상단에 다음 버튼들이 보였음:
  - "DEV: 전체퇴원" (환자 일괄 퇴원)
  - "DEV: 환자추가" (더미 환자 한 명 생성)
- 작업환경만 다른 PC/폴더로 옮긴 뒤, 동일한 코드베이스로 앱을 실행했는데 위 두 버튼이 더 이상 보이지 않음.
- 코드를 수정한 것은 아니며, 환경만 바뀐 상태임.

[요청]
1. 이 프로젝트 저장소에서 위 두 버튼이 **어디서, 어떤 조건으로** 렌더링되는지 정확히 찾아줘.
2. "환경만 옮겼을 때" 그 조건이 거짓이 될 수 있는 구체적인 원인(예: 환경 변수, 빌드 시점, 설정 파일 경로 등)을 나열해줘.
3. 새 환경에서 다시 버튼이 보이도록 하려면 어떤 설정을 어떻게 하면 되는지, 단계별로 안내해줘.

[참고]
- Next.js 사용. NEXT_PUBLIC_* 환경 변수는 빌드 시점에 인라인된다는 점을 고려해줘.
- 프론트엔드 루트는 frontend/, 스테이션 페이지는 frontend/src/app/station/ 쪽일 가능성이 높음.
```

---

## 요약 (에이전트에게 넘기지 않고 빠르게 확인할 때)

- **노출 조건**: `frontend/src/app/station/page.tsx` 51라인 근처에서  
  `process.env.NEXT_PUBLIC_ENABLE_DEV_UI === 'true'` 일 때만 "DEV: 전체퇴원", "DEV: 환자추가" 버튼이 렌더됨.
- **환경만 옮겼을 때 사라지는 흔한 이유**  
  - 새 환경의 `frontend/.env.local`에 `NEXT_PUBLIC_ENABLE_DEV_UI=true`가 없거나,  
  - 이전에 `.env.local`을 복사하지 않아서 값이 비어 있음.  
  - Next.js는 `NEXT_PUBLIC_*`를 **빌드 시점**에 박기 때문에, `.env.local`을 나중에 바꿔도 이미 빌드된 앱에는 반영되지 않음. **개발 서버 재시작 또는 재빌드** 필요.
- **해결**:  
  1. `frontend/.env.local`에 `NEXT_PUBLIC_ENABLE_DEV_UI=true` 추가 또는 수정.  
  2. `npm run dev` 또는 `npm run tauri dev`를 한 번 종료한 뒤 다시 실행해 빌드/번들이 다시 만들어지게 함.
